## 一、更名说明

本项目由原 **DAO（道）** 更名为 **SunDAO（深道）**。

- **DAO（道）**：项目名称源于道家哲学"道生一，一生二，二生三，三生万物"，体现 Being + Type + Relation 的核心三元组设计哲学。
- **SunDAO（深道）**："深"寓意深度（Deep）、深耕（Deep-rooted）、深智（Deep Intelligence），与项目从数据基座向AI认知-应用平台的演进方向一致。

更名后，原 dao 单仓架构拆分为多子项目独立仓库，SunDAO 作为总项目统一协调。原 dao 项目的历史设计文档、架构决策记录仍具参考价值，已归档保留。

## 二、项目定位

**深道（SunDAO）** 是下一代**本体论优先的去中心化AI认知-应用平台**。

为AI Agent提供可持久化、可溯源、可推理的语义数据基座；
为企业与开发者提供"一次本体定义，自动生成完整应用"的动态应用平台；
为去中心化网络提供数据主权与跨组织协作的基础设施。

- 🌐 **去中心化语义网络** — 数据主权回归用户，跨组织AI协作
- 🧠 **海量精准记忆系统** — 七层认知模型，持久化AI记忆
- 🧮 **精确运算推理引擎** — 可验证的数学求解与逻辑推理
- 🎨 **动态应用平台** — 数据定义即界面，def+template+client_script=完整应用
- 🔗 **统一语义层** — Being + Type + Relation 核心三元组

> **我们不是另一个数据库，而是AI与世界的"认知接口"。**
> **我们不只是基础设施，更是让数据自动变成应用的平台。**

## 三、三大核心特征

### 特征一：🌐 去中心化（Decentralization）

**核心理念**：数据主权 + 去信任共识 + 开放协作

**技术实现**：
- **P2P网络层**：QUIC协议 + GossipSub消息传播
- **共识机制**：TempoDAG（时序DAG）+ PoEf（有效性证明）
- **数据确权**：Being级数字所有权，Ed25519密码学证明
- **跨链互操作**：LLM矿池侧链，去中心化推理市场

**价值主张**：

| 维度 | 中心化方案 | SunDAO去中心化 |
|------|-----------|---------------|
| 数据主权 | 平台所有 | 用户所有 |
| 隐私保护 | 信任平台 | 密码学保证 |
| 服务可用性 | 单点故障 | 分布式冗余 |
| AI协作 | 封闭生态 | 开放网络 |

### 特征二：🧠 海量精准记忆（Massive Precise Memory）

**核心理念**：七层认知模型 + 分层存储策略 + 精准检索

```
Layer 7: 元记忆 (Meta)        — 关于记忆的记忆
Layer 6: 程序记忆 (Procedural) — 技能/模式/工作流
Layer 5: 语义记忆 (Semantic)   — 知识/概念/事实
Layer 4: 情节记忆 (Episodic)   — 事件/经历/对话历史
Layer 3: 工作记忆 (Working)    — 会话上下文/短期缓存
Layer 2: 感知记忆 (Sensory)    — 原始输入/瞬时感知
Layer 1: 感官输入 (Input)      — 多模态原始数据
```

**核心能力**：

| 能力 | 说明 | 技术实现 |
|------|------|----------|
| **记忆编码** | 多模态数据→Being | 自动embedding + PII检测 |
| **记忆存储** | 分层存储策略 | SunDaoQL多引擎自动路由 |
| **记忆检索** | 混合检索 | 向量+关键词+图谱+时序 |
| **记忆巩固** | 短期→长期转化 | 重要性评分 + rehearsal |
| **记忆遗忘** | 智能清理 | TTL + 重要性衰减 |
| **记忆推理** | 知识图谱推理 | 多跳推理 + 因果链发现 |

### 特征三：🧮 精确运算与推理（Precise Computation & Reasoning）

**核心理念**：可验证计算 + 符号推理 + 神经语义融合

**三层推理架构**：

| 层级 | 能力 | 代表技术 |
|------|------|----------|
| **L1: 数学精确计算** | 排程优化、约束求解、符号计算 | OR-Tools CP-SAT、Z3 SMT、SymPy |
| **L2: 符号逻辑推理** | 知识图谱多跳推理、因果链发现 | 图遍历、传递/对称推理 |
| **L3: 神经语义推理** | LLM推理、向量相似度、语义匹配 | SunDaoLLM本地推理、向量搜索 |

## 四、从DAO到SunDAO的架构演进

### 第一阶段：DAO（多存储适配器架构）

原DAO项目采用**8存储组件**架构：

```
PostgreSQL + MongoDB + Redis + Neo4j + ClickHouse + Qdrant + ElasticSearch + MinIO
```

每个组件解决特定问题，但组合后产生了远超8倍的复杂度：CDC同步、Saga补偿、查询路由、降级熔断等多套胶水代码。

### 第二阶段：DaoQL（统一存储内核）

从8组件走向**单一自研存储内核**：

```
ACID图引擎 + 列引擎 + 向量索引 + 全文索引 = 单一Rust进程，零外部依赖
```

DaoQL的核心洞察：**"需要建索引的字段都在图上和向量索引上，不需要建索引的都在列库上。"**

### 第三阶段：SunDAO（多子项目生态）

将DaoQL单仓按职责拆分为独立子项目，形成松耦合生态：

| 演进方向 | 说明 |
|----------|------|
| **存储层独立** | SunDaoQL专注核心引擎，成为可复用基座 |
| **服务层独立** | SunDaoQLServer提供网络服务，适配多种部署场景 |
| **AI层独立** | SunDaoAgent + SunDaoLLM专注AI能力，开放trait接入 |
| **客户端独立** | SunDaoQLWeb + SunDaoApp专注交互体验 |

## 五、典型应用场景

### 场景1：去中心化AI助手网络

**问题**：个人AI助手数据被平台垄断，无法跨服务迁移

**SunDAO方案**：
- 用户的知识记忆、对话历史、技能模式以Being形式存储在个人节点
- 用户持有私钥，数据主权归用户
- 跨平台同步：手机App、桌面App、Web统一访问同一Being宇宙

### 场景2：企业级精准记忆系统

**问题**：企业AI项目"失忆"，每次对话都从零开始

**SunDAO方案**：
- 客户记忆图谱：客户A→咨询产品X→投诉订单Y→客服B服务
- 业务知识库：产品规格（语义记忆）、故障案例（情节记忆）、解决方案（程序记忆）
- AI理解"客户A是VIP，咨询过产品X，对订单Y有投诉"，服务连续性提升10倍

### 场景3：智能制造优化求解

**问题**：生产排程复杂，多约束条件下难以找到最优解

**SunDAO方案**：
- 设备Being（状态/产能/维护）、工单Being（优先级/交期/工艺）、物料Being（库存/供应商/质量）
- OR-Tools CP-SAT排程优化 + 知识图谱工艺路线推荐 + LLM自然语言指令理解
- 排程效率提升40%，交期达成率从75%提升到92%
