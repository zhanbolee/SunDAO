## 一、概述

### 1.1 什么是行业模板

SunDAO 行业模板是**基于 def/type/relation 语义预制的行业解决方案包**。它不是传统软件中的"配置模板"，而是一套完整的本体定义（Being + Type + Relation）+ 动态 UI 模板（Template DSL）+ 业务逻辑脚本（client_script）的组合。

**使用方式**：
```
客户选择"汽车零部件行业模板"
    ↓
SunDAO 自动创建：
    - 200+ Being 定义（产品、BOM、工单、设备、物料...）
    - 50+ Type 定义（可制造、可质检、可出库...）
    - 100+ Relation 定义（产品→BOM、工单→设备、订单→出库...）
    - 30+ UI 模板（产品管理、工单看板、质检录入、库存查询...）
    - 20+ client_script（价格计算、排程逻辑、质检判定...）
    ↓
客户基于模板快速定制（改字段、加关系、调界面）
    ↓
6 周上线（传统开发需要 6-12 个月）
```

### 1.2 核心优势

| 维度 | 传统行业软件 | SunDAO 行业模板 |
|------|------------|----------------|
| 实施周期 | 6-12 个月 | **6-8 周** |
| 定制成本 | ¥50-200 万 | **¥10-30 万** |
| 升级方式 | 供应商发版，客户被动等待 | **模板热更新，数据驱动界面自动刷新** |
| AI 能力 | 无或外挂 | **内置 Agent，自然语言操作** |
| 数据追溯 | 依赖数据库日志 | **存储层原生不可变版本链** |

## 二、第一个行业模板：智能工厂（制造业）

### 2.1 模板定位

**目标行业**：汽车零部件、电子制造、机械加工  
**核心场景**：从报价→设计→采购→生产→质检→仓储→物流的全链路管理  
**发布目标**：2028 Q1（商业化启动阶段）

### 2.2 本体定义（Ontology）

#### 核心 Being 定义

```dsl
// ── 产品族 ──
def Product {
    schema {
        name: String                  // 产品名称
        code: String                  // 产品编码
        category: String              // 产品类别
        version: Number               // 版本号（SunDAO 自动管理）
        status: Enum("DRAFT", "ACTIVE", "OBSOLETE")
        unit: String                  // 计量单位
        standard_cost: Number         // 标准成本
        selling_price: Number         // 销售单价
        description: Text             // 产品描述
        image_url: String             // 产品图片
    }
    template { /* 产品管理界面 */ }
    styles { /* 产品卡片样式 */ }
    client_script {
        fn on_mount() {
            // 加载产品历史版本
            this.state.versions = await $dao.query()
                .scanByDef("Product")
                .filter({ code: this.props.code })
                .orderBy("version", "desc")
                .execute()
        }
        fn calculate_margin() {
            return (this.state.selling_price - this.state.standard_cost) / this.state.selling_price
        }
    }
}

// ── BOM（物料清单）──
def BOM {
    schema {
        product_id: BeingId            // 父产品
        version: Number                // BOM 版本
        status: Enum("DRAFT", "APPROVED", "EFFECTIVE", "OBSOLETE")
        created_by: BeingId
        approved_by: BeingId
    }
    template { /* BOM 树形界面 */ }
    client_script {
        fn on_approve() {
            // BOM 审批后自动生成新版本 Product
            $dao.mutate("create_product_version(...)")
        }
        fn calculate_total_cost() {
            // 递归计算 BOM 总成本
            let components = this.getRelated("HAS_COMPONENT")
            return components.reduce((sum, c) => sum + c.cost * c.quantity, 0)
        }
    }
}

// ── 销售订单 ──
def SalesOrder {
    schema {
        order_no: String               // 订单编号
        customer_id: BeingId           // 客户
        order_date: DateTime           // 下单日期
        delivery_date: DateTime        // 交付日期
        status: Enum("PENDING", "CONFIRMED", "PRODUCING", "SHIPPED", "COMPLETED", "CANCELLED")
        total_amount: Number           // 订单总金额
        priority: Enum("LOW", "NORMAL", "HIGH", "URGENT")
        notes: Text
    }
    template { /* 订单管理界面 */ }
    client_script {
        watch state.status(new_val, old_val) {
            if (new_val === "CONFIRMED" && old_val === "PENDING") {
                // 订单确认后自动生成生产工单
                $dao.mutate("create_work_orders_from_sales_order(...)")
                // 自动检查库存
                $server.call({ method: "check_inventory_availability" })
            }
        }
        async fn get_delivery_progress() {
            let work_orders = await $dao.query()
                .scanByDef("WorkOrder")
                .filter({ sales_order_id: this.id })
                .execute()
            let completed = work_orders.filter(wo => wo.status === "COMPLETED").length
            return completed / work_orders.length
        }
    }
}

// ── 生产工单 ──
def WorkOrder {
    schema {
        wo_no: String                  // 工单编号
        sales_order_id: BeingId        // 关联销售订单
        product_id: BeingId            // 生产产品
        quantity: Number               // 生产数量
        planned_start: DateTime        // 计划开工
        planned_end: DateTime          // 计划完工
        actual_start: DateTime         // 实际开工
        actual_end: DateTime           // 实际完工
        status: Enum("PENDING", "RELEASED", "RUNNING", "PAUSED", "COMPLETED", "SCRAPPED")
        priority: Number               // 优先级（数值越小越高）
    }
    template { /* 工单看板界面 */ }
    client_script {
        fn start_production() {
            this.set_state({ status: "RUNNING", actual_start: new Date() })
            // 自动占用物料库存
            $dao.mutate("reserve_materials(...)")
            // 通知 AGV 搬运物料到工位
            $server.call({ method: "agv_deliver_materials" })
        }
        fn complete_production() {
            this.set_state({ status: "COMPLETED", actual_end: new Date() })
            // 自动创建质检任务
            $dao.mutate("create_quality_check(...)")
            // 自动创建入库单
            $dao.mutate("create_inbound_order(...)")
        }
    }
}

// ── 质检记录 ──
def QualityCheck {
    schema {
        work_order_id: BeingId         // 关联工单
        inspector_id: BeingId          // 检验员
        check_type: Enum("INCOMING", "IN_PROCESS", "FINAL", "SHIPMENT")
        result: Enum("PASS", "FAIL", "CONCESSION")
        defect_count: Number            // 不良数量
        defect_type: String             // 不良类型
        check_date: DateTime
        conclusion: Text
    }
    template { /* 质检录入界面 */ }
    client_script {
        watch state.result(new_val) {
            if (new_val === "FAIL") {
                // 质检不合格 → 自动创建返工工单
                $dao.mutate("create_rework_order(...)")
                // 通知质量经理
                $server.call({ method: "send_alert", params: { level: "HIGH", type: "QUALITY_FAIL" } })
                // 自动记录到质量事件 Being
                $dao.mutate("create_quality_event(...)")
            }
        }
    }
}

// ── 设备 ──
def Equipment {
    schema {
        asset_no: String               // 资产编号
        name: String                   // 设备名称
        model: String                  // 型号
        location: String               // 所在位置
        status: Enum("IDLE", "RUNNING", "DOWN", "MAINTENANCE", "RETIRED")
        current_task: BeingId           // 当前执行的工单
        oee: Number                    // 设备综合效率
        maintenance_due: DateTime      // 下次维护日期
    }
    template { /* 设备监控界面 */ }
    client_script {
        watch state.status(new_val) {
            if (new_val === "DOWN") {
                // 设备故障 → 自动调度备用设备
                $server.call({ method: "reschedule_to_backup_equipment" })
                // 通知维护团队
                $server.call({ method: "create_maintenance_ticket" })
            }
        }
        async fn get_oee_trend() {
            let history = await $dao.query()
                .scanByDef("EquipmentStatusLog")
                .filter({ equipment_id: this.id })
                .orderBy("timestamp", "desc")
                .limit(30)
                .execute()
            return history.map(h => ({ date: h.timestamp, oee: h.oee }))
        }
    }
}

// ── 物料/库存 ──
def Material {
    schema {
        code: String                   // 物料编码
        name: String                   // 物料名称
        specification: String          // 规格型号
        category: String               // 物料类别
        unit: String                   // 单位
        safety_stock: Number           // 安全库存
        current_stock: Number          // 当前库存
        warehouse_location: String     // 库位
        batch_no: String               // 批次号
    }
    template { /* 库存管理界面 */ }
    client_script {
        watch state.current_stock(new_val) {
            if (new_val <= this.state.safety_stock) {
                // 库存低于安全线 → 自动创建采购申请
                $dao.mutate("create_purchase_request(...)")
            }
        }
        fn get_inventory_age() {
            // 计算库存周转天数
            let inbound = this.getRelated("INBOUND")
            let outbound = this.getRelated("OUTBOUND")
            return calculate_turnover_days(inbound, outbound)
        }
    }
}
```

#### 核心 Type 定义

```dsl
type Traceable {
    schema {
        batch_no: String
        serial_no: String
        manufacture_date: DateTime
        expiry_date: DateTime
    }
    client_script {
        fn trace_full_lifecycle() {
            // 追溯从原材料到成品的全生命周期
            return $dao.query()
                .being(this.id)
                .withRelations(["RAW_MATERIAL_OF", "PRODUCED_BY", "INSPECTED_BY", "STORED_AT", "SHIPPED_IN"])
                .execute()
        }
    }
}

type Schedulable {
    schema {
        priority: Number
        earliest_start: DateTime
        latest_end: DateTime
        required_capacity: Number
    }
    client_script {
        async fn optimize_schedule() {
            let result = await $server.call({
                method: "solver_optimize",
                params: { tasks: this.getRelated("SCHEDULED"), constraints: this.state }
            })
            this.set_state({ planned_start: result.start, planned_end: result.end })
        }
    }
}

type QualityControlled {
    schema {
        inspection_standard: String
        aql_level: String
        sampling_plan: String
    }
    client_script {
        fn determine_sampling_size() {
            // 根据 AQL 标准确定抽样数量
            return aql_table_lookup(this.state.aql_level, this.props.batch_size)
        }
    }
}

### 2.4 AI Agent 场景（自然语言操作）

```
用户: "下周要交付的订单，生产进度怎么样了？"

Agent 理解意图 → 生成 DSL:
    query {
        SalesOrder(filter: { delivery_date: { lte: "next_week" } }) {
            wo: HAS_WORK_ORDER {
                status, completion_rate, equipment { name, status }
            }
        }
    }

Agent 返回:
    "下周有 12 个订单待交付：
     - 8 个正在生产，平均完成度 65%
     - 3 个已完成，待质检
     - 1 个（SO-2026-0892）因设备 E-012 故障暂停，建议调度到 E-008"
```

```
用户: "A 仓库里有多少批 2026 年 3 月入库的螺丝？"

Agent 理解意图 → 生成 DSL:
    query {
        Material(filter: { name: { contains: "螺丝" } })
            → INBOUND_LOG(filter: { inbound_date: { gte: "2026-03-01", lte: "2026-03-31" } })
            → WAREHOUSE_LOCATION(filter: { zone: "A" })
    }

Agent 返回:
    "A 仓库共有 5 批 2026 年 3 月入库的螺丝：
     - 批次 B-20260315-001：M4×20，数量 5,000，库位 A-03-12
     - 批次 B-20260322-003：M5×25，数量 3,200，库位 A-05-08
     ..."

## 四、技术实现路径

### 4.1 模板引擎架构

```
行业模板包（.sundao-template）
    ├── ontology/
    │   ├── beings/           # Being 定义文件（.def）
    │   ├── types/            # Type 定义文件（.type）
    │   └── relations/        # Relation 定义文件（.rel）
    ├── templates/
    │   ├── pages/            # 页面模板（JSON DSL）
    │   ├── components/       # 组件模板（JSON DSL）
    │   └── forms/            # 表单模板（JSON DSL）
    ├── scripts/
    │   ├── client/           # client_script（.cs）
    │   └── server/           # server_script（.rhai）
    ├── styles/
    │   └── theme.css         # 主题样式
    ├── data/
    │   └── seed.json         # 演示数据
    └── manifest.json         # 模板元数据（名称、版本、依赖、定价）
```

### 4.2 模板安装流程

```
1. 客户在 SunDaoApp 中选择"智能工厂模板"
2. 系统从模板市场下载 .sundao-template 包
3. 解析 manifest.json，检查依赖（SunDaoQL 版本、SunDaoAgent 版本）
4. 安装 Being/Type/Relation 定义到 SunDaoQL
5. 安装 UI 模板到 SunDaoApp 组件注册表
6. 安装 client_script/server_script
7. 可选：导入 seed.json 演示数据
8. 客户基于模板进行定制（改字段、调界面、加逻辑）
```

### 4.3 模板版本管理

```
智能工厂模板
    ├── V1.0.0 (2028 Q1)   # 初始发布
    ├── V1.1.0 (2028 Q2)   # 增加设备预测维护
    ├── V2.0.0 (2028 Q4)   # 增加 AI 排程
    └── V3.0.0 (2029 Q2)   # 增加数字孪生

升级机制：
- 补丁版本（V1.0.1）：bug 修复，自动热更新
- 小版本（V1.1.0）：功能增加，客户选择是否升级
- 大版本（V2.0.0）：架构变更，需要数据迁移
```

### 5.1 模板定价策略

| 模板类型 | 定价模式 | 价格 | 包含内容 |
|---------|---------|------|---------|
| **基础模板** | 免费 | ¥0 | 核心 Being + 基础 UI（开源/社区版） |
| **标准模板** | 一次性授权 | ¥10-30 万 | 完整模板 + 3 个月支持 |
| **专业模板** | 年度订阅 | ¥15-25 万/年 | 标准版 + 持续更新 + 技术支持 |
| **定制模板** | 按项目 | ¥30-100 万 | 基于标准模板深度定制 |

### 5.2 模板市场抽成（生态成熟阶段）

```
第三方开发者发布模板
    ├── 定价：开发者自定
    ├── 平台抽成：20-30%
    ├── 开发者收入：70-80%
    └── 平台提供：模板审核、支付结算、版本托管、用户评价
```
