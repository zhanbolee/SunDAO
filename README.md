## 🌟 项目定位

**深道（SunDAO）** 是**世界模型驱动的智能基础设施**。

不是数据库。不是 LLM。不是低代码平台。  
是一种全新的智能架构——其中**世界模型（Being/Type/Relation）是智能的核心**，**LLM 是推理的基础设施**，**记忆是永恒的积累**，**审计是内生的属性**。

**核心三元组：Being（实体）+ Type（类型）+ Relation（关系）**。一切数据皆实体，一切实体皆有身份、有状态、有历史、有关系。

> **我们不是另一个数据库，而是AI与世界的"认知接口"。**
> **我们不只是基础设施，更是让数据自动变成智能的平台。**
> **我们不只是存储数据，而是在存储一个可以自我演化的世界模型。**

## 🧠 核心设计理念

### 范式一：世界模型是智能的核心，LLM 是推理的基础设施

```
传统架构：              SunDAO 架构：
智能 = 模型权重         智能 = 世界模型（数据结构 + 规则 + 关系）+ 模型（生成能力）
数据 = 被动输入         数据 = 主动语义（自带行为定义）
学习 = 梯度下降         学习 = 写入新概念 + 更新信念度
记忆 = 上下文窗口       记忆 = 推理缓存持久化为 Being（跨会话、跨设备、终身）
审计 = 外部日志         审计 = 原生审计日志（物理不可篡改）
```

### 范式二：一个原语 Being（实体）

SunDAO 中的一切都是实体。客户是实体，订单是实体，「属于」「创建」「相似」也是实体（关系实体）。

每个实体自带 26 个通用业务字段：身份、分类、状态、版本、审计时间、操作人、租户、权限组、搜索编码……**你不再需要为每个表重复设计这些字段。**

### 范式三：记忆即不可变追加

实体不会"被修改"——只会"产生新版本"。旧版本仍在，只是不再是"当前"。

```rust
// 当前状态
sundaoql.query().being(order_id).fetch_one();

// 三个月前的状态
sundaoql.query().being(order_id).as_of("2025-01-15T00:00:00Z").fetch_one();
```

没有审计表，没有 CDC，没有日志管道。**存储层本身就是完整的历史。**

### 范式四：推理与审计的同一事务

LLM 的每次推理步骤和审计记录在同一个事务中提交：

```
BEGIN TRANSACTION
    ├── 更新推理缓存层 (Being)
    ├── READ Expert_47 (Being)        ← 审计：读取了哪些专家
    ├── UPDATE Session.tokens[1423]    ← 生成的 token
    ├── 插入推理链         ← 完整推理路径
    ├── 插入审计记录           ← 审计记录
    └── COMMIT

如果任何一步失败：全部 ROLLBACK
如果崩溃：从事务日志恢复，状态完全一致
篡改任意记录：后续所有记录哈希不匹配 → 检测出篡改
```

## 🚀 快速开始

### 构建核心引擎

```bash
cd SunDaoQL
cargo build --workspace --release
cargo test --workspace
```

### 启动服务层

```bash
cd SunDaoQLServer
cargo run --bin sundaoql-server -- --config sundaoqlserver.toml
```

### 启动Web前端

```bash
cd SunDaoCode
npm install
node server.js
```
